# Instructions for Generating New Blog Articles — SOLON Agency

## Example Prompt to Give the Agent

```
Similarly to the articles in blog/posts/, write a new article called
"Cum sa folosesti [TOOL] ca avocat" where you share specific
functionalities, settings, power uses, tips & tricks, and advanced
configurations that lawyers can use with this digital tool.
Be correct, precise and specific. Write in Romanian and use diacritics.
Generate only the .md file in blog/posts/.
Use the following images inside the article:
- assets/img/[image1.png]
- assets/img/[image2.png]
After creating the .md file, run node scripts/humanize.js,
then node scripts/build.js, then node scripts/update-sitemap.js.
```

## Overview

This is a static website for SOLON Agency, a digital agency for lawyers. The blog system uses Markdown source files that are compiled into HTML via a build script. All articles are written in **Romanian**.

---

## Blog System Architecture

```
blog/
  posts/                         ← Markdown source files (one per article)
    cum-sa-folosesti-gmail-ca-avocat.md
    cum-sa-folosesti-trello-ca-avocat.md
    ...
  cum-sa-folosesti-gmail-ca-avocat/
    index.html                   ← Compiled output (generated by build.js)
  posts.json                     ← Auto-generated index of all posts
  index.html                     ← Blog listing page (auto-generated)

scripts/
  build.js                       ← Compiles all .md files to HTML
  update-sitemap.js              ← Regenerates sitemap.xml

templates/
  layout.html                    ← HTML shell with header/footer
  post.html                      ← Article body template
  list.html                      ← Blog listing template

assets/img/                      ← All images used in articles
sitemap.xml                      ← Updated automatically after build
```

The **only file you need to create manually** is the `.md` file in `blog/posts/`. Everything else is generated by running `node scripts/build.js`.

---

## Step 1 — Create the Markdown File

Create a new file at:

```
blog/posts/[slug].md
```

Where `[slug]` is the URL-friendly article identifier (lowercase, hyphens, no diacritics). Example: `cum-sa-folosesti-slack-ca-avocat`.

---

## Step 2 — Frontmatter Format

Every `.md` file must begin with a YAML frontmatter block enclosed in `---`. All fields are required unless noted.

```yaml
---
title: "Cum să folosești [Instrument] ca avocat"
date: "YYYY-MM-DD"
slug: "cum-sa-folosesti-[instrument]-ca-avocat"
description: "Ghid practic pentru avocați: [2-3 key topics separated by em dashes]. Max 160 characters."
read_time: [number]
categories: ["digitalizare", "productivitate", "automatizări"]
tags: ["[instrument]", "avocați", "productivitate", "[topic1]", "[topic2]"]
---
```

### Field rules

| Field | Rules |
|---|---|
| `title` | Romanian, with diacritics. Format: `"Cum să folosești X ca avocat"` |
| `date` | ISO 8601 format `YYYY-MM-DD`. Use today's date. |
| `slug` | Must match the filename (without `.md`). No diacritics, lowercase, hyphens only. Must also match the folder name that `build.js` will create under `blog/`. |
| `description` | Romanian. Under 160 characters. Used in `<meta name="description">` and the blog card on the listing page. |
| `read_time` | Integer (minutes). Rule of thumb: 200 words/minute. If omitted, `build.js` auto-calculates it. |
| `categories` | Array of strings. Choose from existing: `"digitalizare"`, `"productivitate"`, `"automatizări"`, `"legaltech"`, `"management"`, `"calendar"`, `"securitate"`, `"seo"`, `"marketing juridic"`. |
| `tags` | Array of strings (lowercase). Always include `"avocați"`. Always include the tool name. Max 7 tags. |

## Step 3 — Article Content Rules

### Language and tone

- All content in **Romanian**, with correct diacritics (ă, â, î, ș, ț).
- Tone: professional, direct, practical. No marketing fluff.
- Address the reader as "tu" (informal singular), not "dumneavoastră".
- Be **correct, precise and specific**: name exact menu paths, keyboard shortcuts, setting names, version-specific features. Avoid vague statements like "you can configure various options".

### Structure

- Start with a 2–3 sentence introduction that explains what the tool is, why it matters for lawyers, and what the article covers.
- Use numbered H2 headings (`## 1. Title`) for all main sections.
- Minimum **10 sections**. Aim for 11–15 for a thorough guide.
- End with a **Concluzie** section that summarises the value, names a limitation or trade-off, and closes with a call-to-action referencing **SOLON**.

### Required sections (adapt names to the tool)

1. **Configurare de bază** — initial setup, professional settings, first-run configurations
2. **Organizare / structură** — how to organise the tool for legal work (folders, boards, notebooks, etc.)
3. **Clasificare vizuală** — labels, categories, colour coding, tags
4. **Automatizări** — built-in automation rules, macros, scheduled actions
5. **Funcții avansate de căutare** — search operators, saved searches, filters
6. **Șabloane și conținut reutilizabil** — templates, snippets, reusable blocks
7. **Integrare cu alte instrumente** — Google Calendar / Gmail / Drive / Teams / Zapier etc.
8. **Colaborare și delegare** — sharing, permissions, team access
9. **Securitate și conformitate** — mandatory security settings for legal data (GDPR, client confidentiality)
10. **Aplicația mobilă** — mobile-specific features and tips
11. **Tips & tricks** — at least 7 concrete shortcuts or non-obvious features with keyboard shortcuts where applicable
12. **Concluzie** — summary + trade-off + persuasive creative SOLON Call to Action

### Formatting inside content

- Use `**bold**` for UI element names, menu paths, button names, and key concepts.
- Use `` `code` `` for keyboard shortcuts, command paths, search operators, field names, and values.
- Use bullet lists for feature enumerations. Use numbered lists only for sequential steps.
- Use **tables** for comparison schemas (e.g. colour/category mapping tables).
- Use **fenced code blocks** (` ``` `) for folder/board structure diagrams and multi-line examples.
- Do NOT use inline styles, HTML comments, or custom CSS classes in the content body.

### SOLON call-to-action (mandatory in Concluzie)

Always end the Concluzie with a sentence in this format:

> Dacă dorești să [implement/configure/integrate] [tool] pentru cabinetul tău [adaptat specificului activității / cu automatizări / cu integrare cu Gmail și Calendar], echipa **SOLON** [poate construi / oferă consultanță de digitalizare / poate ajuta cu consultanță] adaptată specificului practicii tale juridice.

## Step 4 — Inserting Images

### Image path convention

Images in article content must use this path format, which is **relative to the compiled HTML output location** (`blog/[slug]/index.html`):

```
../../assets/img/FILENAME.ext
```

### Image HTML block

Always wrap images in this exact Bootstrap layout block:

```html
<div class="row justify-content-center my-4">
  <div class="col-md-9">
    <img src="../../assets/img/FILENAME.ext" alt="Ilustrație: [descriptive alt text in Romanian]" class="img-fluid rounded shadow-sm" loading="lazy" decoding="async" />
  </div>
</div>
```

### Image placement rules

- Place the **first image immediately after the intro paragraph**, before the first H2 section.
- Place **subsequent images mid-article** at natural breaks between sections — not clustered together. Space them at least 4–5 sections apart.
- Use `col-md-8` for wider/landscape images, `col-md-9` for portrait/square images.
- `alt` text must describe the illustration in Romanian and relate to the article's legal context.

## Step 5 — After Creating the .md File

First, run the humanize script to normalise typography in all source files (e.g. replaces em dashes `—` with hyphens `-`):

```bash
node scripts/humanize.js
```

Then run the build script to compile all `.md` files into HTML and regenerate `posts.json` and `blog/index.html`:

```bash
node scripts/build.js
```

This will automatically:
- Create `blog/[slug]/index.html`
- Update `blog/posts.json` (sorted by date descending)
- Regenerate `blog/index.html` (listing page with all articles)

Then update the sitemap:

```bash
node scripts/update-sitemap.js
```

This will add the new article URL (`https://solon.agency/blog/[slug]/`) to `sitemap.xml` with the current timestamp and `priority: 0.80`.
